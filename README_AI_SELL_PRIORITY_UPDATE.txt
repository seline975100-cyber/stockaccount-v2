# AI 股票分析：可賣區間優先邏輯更新

本版修正 AI 股票分析中「明日操作計畫」與「具體建議」可能不一致的問題。

## 修正內容

1. 若目前價格已進入「隔日減碼/賣出區間」，AI 會直接將具體建議改為「先分批賣出 / 減碼」，不再同時建議加碼。
2. 若目前價格已跌入「失守賣出區間」，AI 會將操作優先順序改為「賣出控風險 / 停損」，不再建議攤平。
3. 明日操作計畫中的「可買區間」在賣出情境下會顯示「不加碼」，避免使用者誤判。

## 操作邏輯

實際執行優先順序如下：

1. 失守賣出區間：先賣出控風險
2. 可賣區間：先分批賣出 / 減碼
3. 可買區間：才考慮分批加碼

因此「具體建議」與「明日操作計畫」會保持一致。
